# Current-state claim inventory

This inventory freezes what the tracked repository currently claims in public-facing docs and metadata.
It is factual only: it records source paths, stable identifiers, and currently published wording anchors without implying that any runtime behavior has already been re-proven on live hardware in this slice.

The source tables below freeze where claims are published. The revalidation-cluster tables attach current owner files, expected proof sources, and downstream slice handoffs so later audit work can start from the present tree instead of historical prose.
